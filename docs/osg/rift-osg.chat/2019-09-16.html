(15:42:25) james.clark: 

RIFT is an algorithm for parameter estimation for gravitational wave sources,
using an iterative likelihood-fitting scheme with MCMC sampling to rapidly
evaluate posterior probability distributions in a high-dimensional (15-D)
parameter space.

For the GPU aspect: the likelihood evaluation is easily vectorizable and based only on simple primitives. Plus, it's naturally compressed so it's possible to fit it on a GPU, regardless of the signal duration
(15:42:45) isfiligoi: Why is it important for LIGO science?
(15:43:58) isfiligoi: The audience will know nothing of physics (to the first approximation)
(15:48:21) james.clark: Rapid parameter estimation is important to understand the nature of the source and the potential for multi-messenger emission. In other words, is it worth spending telescope time following up a detection? in an ideal world, we want to catch things like binary neutron star, or neutron star-black hole mergers "in the act" and observe all of the electromagnetic emission from the time of merger, through to the afterglow. There's a lot to learn in the first few minutes immediately following a merger. On the other hand, there isn't very much to look at with a telescope when you have a binary black hole merger - so knowing things like the component masses (from which you have some idea of whether it possibly features a neutron star), and knowing it *quickly* is extremely important. People will happily interrupt observing schedules to point a telescope at a binary neutron star merger -- they'll be a lot less pleased if it turns out to be 2 black holes
(15:49:10) james.clark: so, the faster, the better. Moving to GPUs speeds up that parameter estimation (and what you think the source is) by a factor of about x20
