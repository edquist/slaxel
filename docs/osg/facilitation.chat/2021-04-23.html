(12:54:22) lmichael: Not that I know of, but there ARE steps in some CryoEM analysis pipelines that would be good candidates, based upon discussion with our new CryoEM center at UW-Madison. Some of the common analysis software assumes it’s running on a single server for all workflow steps, but we totally had a researcher at UW-Madison run work on CHTC that could have been on OSG, but was really only several hundreds 1-hour job each time he ran - easy enough for our local resources to chew through in an afternoon, anyway.
(14:05:26) cbrown58: Good to know. Thanks for the info!
(14:09:30) lmichael: Basically, there are steps that have to look at all images in a set, together, and steps that look at each image (e.g. frame, tomogram, etc.) independently. It’s those independent steps that can be really time consuming to run in serial or even just on a single server, and splitting can make a big difference. Especially when there are multiple movies (sets of frames or tomograms) to analyze, HTC-style parallelization makes big difference.
